# Anti-Copy Policy

This showcase is a teaching artifact, not an Assignment 2 template.

## Acceptable reuse

- Reusing the idea of comparing a baseline policy to a learned policy.
- Reusing the idea of a reward hacking check.
- Reusing the idea of an offline evaluation grid.
- Reusing the idea of a deploy-shadow-reject memo.

## Unacceptable reuse

- Copying the environment class.
- Copying the reward function with renamed variables.
- Copying the same state variables.
- Copying the same action set.
- Submitting the student-support domain for Assignment 2.
- Copying the same evaluation report structure with only changed labels.

## Submission expectation

Assignment 2 submissions should include a Domain Delta Statement explaining how the domain, state, actions, reward, and evaluation differ materially from this showcase.

If a submission copies the showcase environment or reward function with only cosmetic renaming, the implementation score should be capped according to course policy.
